잠재적인 문제
	pop, top - 이 둘은 스택의 꼭대기 항목을 조작한다.
	스택이 공백 상황일 때 pop, top은 어떻게 되는가?
	
잠정적인 문제 해결
	1. 메소드 내부에서 자체적으로 문제를 해결.
	2. 상황에 맞게 예외 발생.
	3. 잠정적인 문제에 대한 사전 조건 설정.
	
잠정적인 문제 해결 3번으로 isEmpty 관찰자 메소드를 정의한다면 매번 스택 연산을 수행할 때
마다 관찰자 메소드로 인한 사전 검사를 하게 되며 코드는 비효율적이고 code 가독성이 낮아
진다.

잠정적인 문제 해결 2번으로 생각해 본다면 잠정 3번 보다 상황은 좋아진다.
RuntimeException을 확장하여 사용하며 이러한 예외는 검사되지 않는다.
프로그램에서 명시적으로 감지될 필요가 없다. 명심하자.
